## Lab Report: Introduction to Diffusion Models

### a. Description of the Diffusion Model and Key Components

In this lab, I implemented a Denoising Diffusion Probabilistic Model (DDPM), which is a class of generative models inspired by non-equilibrium thermodynamics. The model learns to generate data by reversing a gradual "noising" process.

The framework consists of two main processes:

1.  **The Forward Process (Diffusion Process):**
    This is a fixed Markov chain that gradually adds Gaussian noise to the data according to a variance schedule ($\beta_t$). As the number of steps $t$ increases, the data $x_0$ is corrupted until it becomes indistinguishable from isotropic Gaussian noise $\mathcal{N}(0, I)$ at step $T$. This process is parameter-free and does not require training.

2.  **The Reverse Process (Denoising Process):**
    This is the learned process where a neural network is trained to reverse the corruption. Starting from pure noise $x_T$, the model iteratively removes noise to reconstruct the original data distribution $x_0$.

**Key Components:**

* **The Neural Network (U-Net):** The core of the model is a U-Net architecture (often with attention mechanisms). Instead of predicting the image directly, the network $\epsilon_\theta(x_t, t)$ is trained to **predict the noise** that was added to the image at a given timestep $t$.
* **Time Embeddings:** Since the U-Net parameters are shared across all timesteps, a sinusoidal time embedding (similar to Transformers) is injected into the network to tell it which step $t$ of the diffusion process it is currently handling (i.e., how much noise to remove).
* **Loss Function:** The model is trained using a simple Mean Squared Error (MSE) loss between the actual noise added (sampled from a normal distribution) and the noise predicted by the U-Net:
    $$L_{simple} = || \epsilon - \epsilon_\theta(\sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\epsilon, t) ||^2$$

### c. Reflection and Take-aways

This lab provided a fascinating insight into how structured data can emerge from pure randomness. My key take-aways are:

1.  **Simplicity of the Objective:** It is surprising that training a network to simply predict "noise" (using a standard MSE loss) allows for such high-quality image generation. The complexity lies in the iterative process rather than the loss function itself.
2.  **Computational Cost:** Compared to previous models like Naive Bayes or simple CNNs, the sampling process in Diffusion Models is computationally expensive because it requires a full pass through the U-Net for every single timestep (e.g., 1000 steps) to generate just one image.
3.  **Stability:** Unlike GANs, which I have read can be unstable due to the adversarial min-max game, the Diffusion Model training felt stable and the loss curve decreased consistently.
